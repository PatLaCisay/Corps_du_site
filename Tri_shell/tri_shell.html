Tri_Shell : l'algorithme

Principe

Le tri de Shell est une amélioration du tri par insertion en observant deux choses :

  • 1 - Le tri par insertion est efficace si la liste est à peu près triée
  • 2 - Le tri par insertion est inefficace en moyenne car il ne déplace les valeurs que d'une position par instruction
    • Le tri de Shell trie chaque liste d'éléments séparés de n positions chacun avec le tri par insertion. L'algorithme effectue plusieurs fois cette opération en diminuant n jusqu'à n=1 ce qui équivaut à trier tous les éléments ensemble. Cette espacement s'appelle le gap.

    Le fait de commencer avec des éléments espacés permet de pallier le deuxième incovénient de l'insertion, tandis que lorsque l'on fait à la fin avec un espacement (ou gap) de 1, ce qui est en fait un tri par insertion ordinaire, on tire parti de l'avantage (1).

    Exemple

    Les éléments permutés sont indiqués en couleur rouge, et les éléments déjà classés en vert.

    1 2 6 4 3 8 Gap : 4
    1 2 6 4 3 8 Gap : 4
    1 2 6 4 3 8 Gap : 3
    1 2 6 4 3 8 Gap : 3
    1 2 6 4 3 8 Gap : 3
    1 2 6 4 3 8 Gap : 2
    1 2 3 4 6 8 Gap : 2
    1 2 3 4 6 8 Gap : 1
    1 2 3 4 6 8

    Code JavaScript 

    			function tri_shell(tableau) {
				gaps=[6, 4, 3, 2, 1];
				for (i=0; i < gaps.length; i++) {
					var gap = gaps[i];
			
					for (var debut = 0; debut < gap; debut ++) {
						tri_insertion(tableau, gap, debut);
					}
				}
			
				return tableau;
			}

    On remarque que nous avons appelé une fonction tri_insertion , il s'agit tout simlement de la fonction de tri par insertion 

    		function tri_insertion(tableau, gap, debut) {
			var en_cours, j;
		
			for (var i = gap + debut; i < tableau.length; i += gap) {
				en_cours = tableau[i];
				j = i;
				while (j > 0 && tableau[j - gap] > en_cours) {
					tableau[j] = tableau[j - gap];
					j -= gap;
					console.log("j="+j+" ; gap="+gap)
				}
		
				tableau[j] = en_cours;
			}
		}

    Complexité

    Le nombre de permutations est tableau.length − 1 (une à chaque étape).

    A chaque étape, trouver le minimum requiert t.length − 1 − i comparaisons.

    En faisant le même calcul que pour le tri à bulles, la complexité de cet algorithme est donc quadratique, dans le cas moyen, dans les cas le meilleur et le pire. Ce tri nécessite moins de permutations que le tri à bulle, et est en pratique plus rapide. Néanmoins, sur un tableau déjà trié ou presque trié, cet algorithme est très mauvais, puisqu’il reste de complexité quadratique. Il n’y a pas comme dans le cas du tri à bulles de façon simple de s’arrêter avant la fin.
